Agent 的"工具调用"在什么情况下是真实计算？什么情况下是查询缓存？分界线在哪？

推荐分层

✓

三层混合模型（推荐）

Layer 🟢 实时 Python：免疫关联、药物筛选 — 数据在 pandas DataFrame 中，scipy.stats 直接算 Spearman/pearson，毫秒级。

Layer 🔵 缓存查询：差异表达、生存分析 — 从预计算 JSON 中查询，Agent 的智能体现在"选什么基因、怎么解释结果"，而非"跑什么统计方法"。

Layer 🟡 降级计算：缓存未命中时 — 简单的 t-test / correlation 用 Python 实时算，复杂分析（Cox）标记"数据不可用"触发 F4 恢复。

关键设计原则

Agent 不区分"真/模拟"——它始终调用同一个工具接口
Agent 调用: tool("survival_analysis", {"gene": "CSTB"})
⬇
工具内部判断: CSTB 在缓存中? → YES → 读缓存 JSON → 返回
                          TP53 在缓存中? → YES → 读缓存 JSON → 返回
                          EGFR 在缓存中? → NO → F4 恢复: "此基因无预计算数据"

Agent 的 智能 体现在：从 LiteratureReview 的 hypotheses 中决定"应该分析什么"，而非"R 代码怎么写"。工具是黑盒。
