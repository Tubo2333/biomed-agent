问题 5：S3 Pipeline 如何实现 EvalAgent Protocol

S2 的 BiomedBenchmark 需要评估 S3。MultiAgentPipeline 必须实现 run(BenchmarkTask) → dict。

核心挑战

S3 的 Pipeline 是一个完整的 4 Agent 串行流程，输入是自然语言问题，输出是报告。但 S2 的 5 个 tasks 期望不同的输入输出格式。

5 Tasks 的适配方案

Task S3 运行方式 适配难度
T1-LIT 只跑 Phase 1（LiteratureAgent），其他 Phase 跳过
→ 就是 S1 的 LiteratureAgent，格式完全一致		 ✅ 简单
T2-GDA 只跑 Phase 1+2（检索+判断关联强度）
→ 输出 association_judgment + evidence		 🟡 中等
T3-DEG 只跑 Phase 2+3（OrchestrationAgent.plan + AnalysisAgent 差异表达工具）
→ 输出 differentially_expressed_genes + method		 🟡 中等
T4-SURV 只跑 Phase 2+3（生存分析工具）
→ 输出 hazard_ratios + km_plot_data		 🟡 中等
T5-DRUG 只跑 Phase 2+3（药物筛选工具）
→ 输出 drug_correlations + fdr_results		 🟡 中等

推荐方案：Task Router + 统一 run() 接口

MultiAgentPipeline.run(task) 逻辑
class MultiAgentPipeline:
    def run(self, task: BenchmarkTask) -> dict:
        task_id = task.task_id

        if task_id == "T1-LIT":
            # 只跑 Phase 1
            review = self.literature_agent.run(task)
            return self._format_as_lit_output(review)

        elif task_id == "T2-GDA":
            # Phase 1 + 简化 Phase 2（不生成完整 DAG）
            review = self.literature_agent.run(task)
            association = self.orchestration_agent.assess_association(review, task)
            return self._format_as_gda_output(association)

        elif task_id in ("T3-DEG", "T4-SURV", "T5-DRUG"):
            # 跳过 Phase 1，直接用 task.input 驱动 Phase 2+3
            plan = self.orchestration_agent.plan_from_task(task)
            results = self.analysis_agent.execute(plan)
            return self._format_as_analysis_output(task_id, results)

    @property
    def name(self) -> str:
        return "MultiAgentPipeline"

关键设计点

  1. T1-LIT 直接委托给 S1 的 LiteratureAgent（已在 S2 评测中跑通）
  2. T3/T4/T5 跳过文献检索 Phase，直接用 GT 中的基因名+数据集构造 plan —— 因为 benchmark 测的是"分析方法选择+执行"，不是文献检索
  3. T2-GDA 是中间态：需要少量文献检索 + 关联判断，不走完整 4 Agent
  4. format 方法（_format_as_*）负责把内部类型映射为 benchmark 期望的 dict 格式
