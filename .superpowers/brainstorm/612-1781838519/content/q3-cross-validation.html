问题 3：Layer 4 交叉验证的具体实现

00B 规定：Agent N+1 不盲目信任 Agent N 的输出。每个下游 Agent 必须运行 validate_upstream()。

三个交叉验证节点的精确逻辑

节点 1: A2 验证 A1 输出

A1 输出：LiteratureReview

📥 A2 的 validate_upstream() 检查：

  1. 证据链内部一致性：扫描 evidence_chain，检测相反方向的 claim（如 "CSTB 与预后相关" vs "CSTB 与预后无关"）→ 标记 conflict
  2. 假设-证据对应：每个 hypothesis 的 rationale 是否引用了 evidence_chain 中实际存在的 claim？→ 没有引用 → WARNING
  3. 置信度合理性：如果 evidence_chain 中 ≥50% 的 claim 是 weak/unverified，但整体 confidence > 0.7 → WARNING
  4. BLOCKER 条件：evidence_chain 为空 或 hypotheses 为空 → BLOCKER，退回 A1

⚠ 注意基因名归一化（D1-01 deferred）：CSTB ≠ cystatin B

节点 2: A3 验证 A2 输出

A2 输出：AnalysisPlan (DAG)

📥 A3 的 validate_upstream() 检查：

  1. 数据源存在性：每个 DAG 节点引用的数据文件路径是否真实存在？→ pathlib.Path.exists()
  2. 基因名有效性：plan 中提到的基因是否在数据列中存在？→ 检查 TCGA 表达矩阵的 rownames
  3. 方法合理性：plan 指定的分析方法是否在可用工具列表中？→ 不在 → WARNING, 降级为最接近的工具
  4. BLOCKER 条件：plan 中所有节点的数据源都不存在 → BLOCKER

核心保护：防止 A2 幻觉出不存在的文件路径或数据列

节点 3: A4 验证 A3 输出

A3 输出：AnalysisResults（每个 DAG 节点的定量结果）

📥 A4 的 validate_upstream() 检查：

  1. 统计量合理性（复用 S2 的 V3 硬规则）：HR 在 0.01-100 间？p-value 在 0-1 间？logFC 在 -20~20 间？
  2. 跨节点矛盾检测：节点 A 说 CSTB 在肿瘤中高表达(logFC>0)，节点 B 的 Cox 说 CSTB 是保护因素(HR<1) → 这不是错误，但需要标记 "needs biological explanation"
  3. 缺失节点检查：plan 中的节点是否都有对应的 result？→ 有缺失 → 列出 "未完成的分析"
  4. BLOCKER 条件：所有分析节点的 result 都为空 → BLOCKER

实现策略

✓

规则为主 + LLM 为辅

规则层（程序化，无 LLM 调用）：数据源存在性、基因名有效性、统计量范围、空值检查、BLOCKER 判定。约 80 行/节点。

LLM 层（仅在边界情况下触发）：跨节点矛盾是否需要生物学解释？证据链矛盾是否是真正的学术争议？仅在 WARNING 级别触发。

实现位置：每个 Agent 类的 validate_upstream(input) -> ValidationReport，返回 PASS / WARNING / BLOCKER。
