Step 3: Multi-Agent Pipeline — 整体架构总览

4 个 Agent 串行协作，完成从文献到报告的完整科研闭环

Pipeline 数据流
👤 User Question: "CSTB 在结直肠癌中的预后价值和免疫浸润关联"
▼
Agent 1: LiteratureAgent (S1 产出)
PubMed 检索 → LLM Rerank → 证据链 → 假设生成
输出: LiteratureReview {evidence_chain, hypotheses, knowledge_gaps}
▼ Layer 4 交叉验证点
Agent 2: OrchestrationAgent (S3 新建)
LLM 驱动动态 DAG 生成 → 方法匹配 → 数据源分配
输出: AnalysisPlan (DAG of analysis tasks)
▼ Layer 4 交叉验证点
Agent 3: AnalysisAgent (S3 新建)
Think→Act→Observe 工具调用 → 失败恢复 F1-F5
输出: AnalysisResults (每个节点的定量结果 + 执行日志)
▼ Layer 4 交叉验证点
Agent 4: ReportAgent (S3 新建)
多源整合 → 假设验证判断 → 结构化报告 → 局限性
输出: Structured Report (.md)
